# Skill: Brainstorming

Shape vague ideas into a clear enough product direction to create a project brief.

## Procedure

1. Capture the raw idea without judging it.
2. Identify the initial user, problem, and desired outcome hypotheses.
3. Narrow the problem before discussing implementation.
4. Identify the primary user and MVP user focus.
5. Explore the simplest useful solution shape.
6. Define MVP scope, non-goals, and first vertical slice.
7. Scan for sensitive areas and approval needs.
8. Identify stack preference or mark it as TBD.
9. Produce a Brainstorming Handoff.

## Rules

- Do not write production application code during brainstorming.
- Do not ask a long questionnaire upfront.
- Ask one high-leverage question at a time.
- Make safe assumptions explicit.
- Do not treat stack choice as product clarity.
- Do not call Implementer during brainstorming.

## Output

Use `ai/09-intake/BRAINSTORMING_PLAYBOOK.md` and `ai/templates/BRAINSTORMING.template.md`.

Minimum final output:

- product idea;
- problem;
- users;
- desired outcome;
- MVP scope;
- non-goals;
- core workflows;
- sensitive areas;
- stack direction;
- first vertical slice;
- recommended workflow mode;
- recommended squad level;
- required next artifacts;
- one blocking question, if any.
