# Skill: Test Planning

Define the minimum reliable verification strategy for a story or phase.

## Procedure

1. Convert each acceptance criterion into one or more checks.
2. Identify automated tests: unit, integration, component, E2E, contract, or snapshot when appropriate.
3. Identify manual QA checks when automation is not enough.
4. Identify edge cases and regression areas.
5. Identify required commands.
6. Write or update `ai/05-execution/TEST_PLAN.md` when non-trivial.

## Output

Include acceptance criterion coverage, test types, commands, manual QA notes, risks, and missing coverage.
